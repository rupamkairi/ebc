item-listing-search
  create a advanced listing search component.
  multi stage component - 
    1. search & select sub category (autocomplete)
    2. search & select brand (autocomplete)
    3. search & select specification (autocomplete)
    4. search & select item (input field)
Use <autocomplete components> 
but above fields (sub category, brand, specification) are not mandatory
 (but item search field text length should be at least 3 characters to init search)
Use this API `POST /item/list`
  - Body:
    ```json
    {
      "search": "String",
      "categoryId": "UUID",
      "brandId": "UUID",
      "specificationId": "UUID",
      "type": "PRODUCT"
    }
    ```

to show item results do not use autocomplete (that is why I created item field normal input) 
show a separate list with (list-item should contain item.name & item.description) a select button next to it.
