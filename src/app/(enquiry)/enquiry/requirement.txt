# Create Enquiry Page.
Buyer details Stage - 
  - User information - Inputs for Name, Phone Number, Email (Optional), Address(Optional) (If there is active user session - prefill these values.)
  - Where do you want to deliver the items? Input for PinCode (Mandatory) [to aslo show preview the state & district]
  - When do you need? Date Selection (Mandatory) [with calendar view that allow to select some later date from current]
  - Add a text area to add remarks (optional).
  - Allow user to add attachments (optional). attachments can be document or medai.

Item selection Stage - 
  - To find & select Items try using components/search/item-search/item-search.tsx.
  - To each item add a quantity input (mandatory) with unit selection (using acceptableUnitTypes) & remarks (optional) input.
  - This way user can add multiple item (line_item) in the enquiry.
  - Labels should be user friendly & Inputs should look good.

Review Stage - (also work as Singl Enquiry Page) (components of this stage would be reused on enquiry-quotations page. (where we show quotations against enquiry))
  - Show all the details of the buyer.
  - Show all the details of the items.
  - Important Note - The components used here would be later Useful.
    - Let me give you more context about the features. There is Enquiry & Quotations. (Do not implement Quotation these now, this is for your context.) 
    - User Creates a Enquiry -> Seller Creates Quotation -> Buyer sees Quotaiton -> Accepts/Rejects/ or Request Revision.

Submit stage (& silent login completion stage)
  - If there is no active user session, then we send Name & Phone to get a OTP on the phone & prompt to verify with the OTP. (this way we will conduct a silent login for the buyer)
  & submit the enquiry. Then navigate to Review Stage

General Information - All the info here should be Drafted into Browser Storage & on Any Unfinised flow would help  user to start from the stored data in the browser.
& whole state should load on browser refresh.
